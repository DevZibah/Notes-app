About This Page

This page is meant to record common errors that one may encounter, and how-tos of some certain things that may be tricky.
Do a "ctrl + F" search for a keyword in the error message you got.

For example, if you're experiencing "error 405: put not supported", press "ctrl + F" and then search for the word "put". That would cause a search through the keywords on the error list and take you to the errors that feature that word. That way, you can find the specific error you want to read about if it's here, without doing much scrolling.

Issue
: for some reason, the data on my list stopped showing when I bound the viewItem.stuff to the view modal. I removed the binding and it showed again.

Solution
: that happened because of "typeScript Error cannot find properties of undefined reading viewItem.stuff". This is because I didn't define the viewItem.stuff specifically in the ts. To avoid this, define the relevant variable in the ts file (and set its value where necessary) it can also be avoided by simply writing viewItem?.stuff.

Issue
: I was trying an edit operation, but I was getting "error 405 put not supported". My code to edit was correct, but I kept getting that error.

Solution
: when you get error 405 "put not supported", it looks like a backend error but it's possible that you didn't write/build the link properly. Copy the actual link from the response body where you tested the endpoint, not from the way the link is written in the documentation and ensure you've built the link well.

Issue
: I was trying to post an image to a backend server in an edit operation, but it kept failing and telling me the content-type must be multipart. I specified that in my service file's post request, but it then asked for the "boundary", which is not something I can specify myself to the best of my knowledge. It's a backend string that the code generates under the hood.

Solution
: Instead of calling the post function in the service file, I did a work around where I made the server request in the ts file.
-- I first captured the image from the upload image event, then in the edit function, I made new FormData and appended the image to it (if the image exists).
-- I then made the put request right there in the ts file, properly capturing the relevant ID and passing in the FormData.
